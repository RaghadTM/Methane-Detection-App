4.3.2.1 Safety Margin and Early-Warning Detection Philosophy 
A key design objective of the final system is operation within the  sub-LEL regime, 
ensuring that methane detection occurs significantly before concentrations approach flammable 
conditions. This transforms the sensing system from a reactive hazard indicator into 
a preventive early-warning mechanism, consistent with petroleum -industry risk-management 
philosophy where mitigation is prioritized prior to escalation. 
Let Cd denote the minimum detectable methane concentration expressed in %LEL.  
Safe early-warning operation requires: 
Cd ≪ 100% LEL  
and practically: 
Cd ≤ 1–5% LEL  
Operating within this conservative range provides: 
• Large safety margin from ignition risk 
• Sufficient time for corrective intervention 
• Reduced probability of fire or explosion escalation 
Thus, the LEL -referenced framework defines both  technical detectability  and safety 
compliance simultaneously. 
4.3.2.2 Methane Dispersion Physics and Distance-Dependent 
Detectability 
Following leakage at a pipeline joint, methane forms a  localized turbulent plume  whose 
concentration decreases with radial distance due to atmospheric dilution and mixing. Although 
full computational fluid-dynamics modeling is outside project scope, dispersion behavior can 
be represented using a simplified power-law decay: 
C(x) ∝ (1/x^n), n > 0 
where: 
 
 
 72 
• x = distance from leak source 
• n = empirical dispersion exponent influenced by turbulence and airflow 
This relationship provides several engineering implications: 
1. Detectability is strongly distance-dependent, requiring sensor placement near probable 
leak sources. 
2. Joint-level monitoring is justified , since methane concentration is highest close to the 
containment failure. 
3. Validation testing must include multiple distances  to confirm consistent detection 
within the expected near-field region. 
These conclusions reinforce the petroleum -engineering decision to focus sensing at  pipeline 
joints rather than along intact pipe segments. 
4.3.2.3 LEL-Based Detectability Envelope and Safety Interpretation 
Combining dispersion physics with LEL normalization allows definition of a  detectability 
envelope bounded by: 
• Upper bound: 
100% LEL → flammable mixture and unacceptable operating condition 
• Lower bound: 
Sensor detection threshold Cd → minimum measurable methane presence

4.3.3.6 Backend API & Real-Time Processing 
The backend is implemented using FastAPI (Python). It performs: 
• Secure HTTPS request handling 
 • Feature extraction 
 • Isolation Forest inference 
 • Anomaly scoring 
 • Alert classification 
 • Database storage 
Server-side inference ensures centralized model control and eliminates the need for firmware 
updates during model improvements. 
 
4.3.3.7 Secure Communication Layer (HTTPS/TLS) 
Data transmission occurs via HTTPS secured using Transport Layer Security (TLS). 
TLS provides: 
• Asymmetric key exchange 
 • Session key derivation 
 • AES-based symmetric encryption 
 • Integrity verification 
According to the TLS 1.3 specification (Rescorla, 2018), AES is used internally for symmetric 
encryption after secure key negotiation. Therefore, manual AES key exchange is unnecessary, 
as TLS handles cryptographic negotiation and encryption automatically. 
This design satisfies system-level encryption requirements while reducing key management 
complexity.

and graded alerts.

4.5  Cost Analysis  
The total estimated cost of the final methane leak detection prototype is 14,769 SAR, which 
satisfies the project constraint of maintaining the total prototype cost below 15,000 SAR. The 
 
 
 85 
cost distribution reflects a deliberate balance between sensing performance, safety 
considerations, and financial feasibility at the laboratory prototype stage.  
A significant portion of the budget is allocated to the optical subsystem, including the 1650 nm 
laser diode, InGaAs photodetector, laser mount, and current controller. These components 
represent the core detection mechanism of the system. Since methane de tection relies directly 
on infrared absorption and precise optical stability, investing in reliable optical hardware was 
necessary to ensure measurement accuracy, signal consistency, and safe laser operation. The 
laser mount and current controller, althoug h relatively high in cost, provide temperature 
stability and controlled excitation, which are essential for maintaining wavelength integrity and 
avoiding signal drift.  
Mechanical components, such as pipeline sections and joints, account for a smaller but essential 
portion of the total cost. These elements enable realistic simulation of joint -level methane 
leakage under controlled laboratory conditions. Their inclusion ensures that experimental valid 
ation reflects practical containment-loss scenarios rather than abstract gas exposure testing. 
The software-related costs, including application deployment and cloud hosting, represent a 
relatively minor fraction of the total budget. However, they significantly enhance the system’s 
functionality. The cloud platform enables remote monitoring and structured data storage, while 
the mobile application provides real -time alerts and visualization. The AI-based classification 
layer further improves reliability by reducing false alarms and enhancing decision support. 
Notably, these software components increase system capability without substantially increasing 
overall cost.  
Overall, the cost structure demonstrates that a technically robust, multidisciplinary methane 
detection prototype can be implemented within the defined academic budget constraints. The 
final cost remains below the specified limit while preserving sensing accuracy, safety integrity, 
and intelligent monitoring functionality, making the system both economically feasible and 
technically credible.
